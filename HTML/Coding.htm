Coding Techniques
Option Explicit
  All Of You Would have seen Option Explicit Statement In The Code Modules
  One Great Benifit For Using It is that you will not make mistakes in
  Calling Variables and it Discards the Possibility Of Mis-Spelling The
  Variable Names. By Placing an Option Explicit Statement You are Telling
  Visual Basic that all the Variables Must Be Declared Before They Are Used.
  Like this If You Mis-Spell any Variable Than VB will Raise an Error.
  This is very Helpful in Bigger Projects Where there are Many Variables
  If You Want VB to Put Option Explicit Statement on Every Form or Module
  Then Click on Tools Menu and Then On Options and On Editor Tab
  Check Require Variable Declaration Box. That's it.
Default Properties
  When We Place any Text-Box or Command-Button There Are Some Properties
  Which Are Highlighted By Default. In Text-Box Its Text Property and In
  Labels Their Caption Properties Are Selected By Default
  This are all Default Properties For These Controls. So If You Write Like This
  strName = txtName Instead of strName = txtName.Text
  Then this Code Will Work and It will also Increase Performance Of Your Application
For...Next Loop
  Some Programmers Always Puts a Counter at the end of the Loop
  But If You Remove that Counter Then Also Your Code Works Fine
  And It is Faster than the For...Next Loop With a Counter at the end.
  OtherWise If Applicable Then Use The Do While or Do Until Loop
If Statement....
  When you are Using If Statement and you only have one Expression
  To be Executed than there is no need to put End If
  Eg:- If a = True Then B = False
Another Way Of Writing If Statement
  When You are Useing The If Else Statement With One Expression To Be
  Executed. Then You can Use IIF Statement Instead Of If Statement
  Eg:- MsgBox IIF(a = 1,b = 2,c = 3)
  The Abouve Statement Is Used Instead Of The Following Code
  If a = 1 Then
  b = 2
  Else
  c = 3
  End If
Using Select Case
  When You Have Many IF's And You Code Goes On and On in this IF Logic
  Than You can Use Select Case Statement
  For Example:- If You Have a Text-Box in Which Your Ten User will
  Input Their Names And You Want To Know Which One Had, Then
  Select Case txtName.Text 'Text-Box's Name
  Case Jack
  lblDisplay.Caption = "Welcome Jack"
  Case Harry
  lblDisplay.Caption = "Welcome Harry"
  Case Anna
  lblDisplay.Caption = "Welcome Anna"
  Case Julie
  lblDisplay.Caption = "Welcome Julie"
  And So On....
  But In Last You Have To Add End Select Like We were Adding End If
