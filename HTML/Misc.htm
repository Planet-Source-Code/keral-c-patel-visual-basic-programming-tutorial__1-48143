Optimizations
Okay So Every Body wants their Programs to Run Faster and Smoother
But Sometimes Our Own Mistakes Makes them Unreliable.
Here are Some tips That Would Help You In its Over all Performance
First Of all always Use $ Functions. Means If You Want to Convert a
Lower Case String to an Upper Case String than Using Ucase$() will
Make Your Program More Efficient. Tests Have Proved that It Reduces the
Execution Time By its Half or its One-Third. You Can Use $ With Many Functions
All String Realted Functions Support This. Format$( ),Chr$( ),Asc$( ),Mid$( )
Hex$( ),Command$( ),Trim$( ),Date$( ),Time$( ) are Some Of the Examples Of it.
As I have Earlier Discussed in Coding Part Never Place Counter at the End of
For...Next Loop. And Use Long as the Counter For it.
Other Things are Like Use 'intX \ 1' Instead of 'Cint(intX)'
Use 'intX * intX' Instead of 'intX ^ 2'
Use 'intX \ 2' Instead of "intX / 2'
Avoid Using Variants WhereEver Possible.
For...Next is Much Faster Than For...Each Loop
ByRef is Also More Efficient than ByVal
Remove all the Code Lines For Debug.Print
Use Control Arrays WhereEver Possible.
Avoid Lot Of Timers as they also Use Memory.
Place DoEvents Statements At Specific Distances.
Always create a Control array of Labels As they
are Seldom Used or Changed within the Code.
Last Thing is That Compile it For Fast-Code or Small-Code
WhichEver is Suitable By Going into Project Properties' Compile Tab.

Common-Dialog's Filters
The CommonDialog control's Filter property lists text strings
alternating with filters separated by pipe | symbols.
Eg:-CommonDialog1.Filter = "JPEG Files (*.jpg)|*.jpg|All Files (*.*)|*.*"
So This will Display a Dialog Box with Two kind of File Filters
First One will be JPEG Files (*.jpg) and Second One will be All Files (*.*)

Optimizing Controls
So here is One more Trick for Optimizing Your Program.
If a situation arises and You want to Add 500 or more Strings
in a ListBox then making its Visible property to False just
Before adding all that data to the ListBox and Making its
Visible Property to True after the strings have been added
Makes Your Program 25 to 30 Percent More Efficient then Under
The Normal Circumstances. The Results may not be Same on
All PC's But On My PC it is 25 to 30 Percent Faster!
