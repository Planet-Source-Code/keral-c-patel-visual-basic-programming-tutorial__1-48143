Variable Declarations
Declaring Variables
  Many Times We Declare Variables Like This
  Dim strName, strAddress, strCity As String
  But Here Only strName will be Declared as String While
  Others Will Be Declared as Variants. As You Know Variants
  Use More Memory This Degrades Your Application's Performance.
  The Right Way Is Like This
  Dim strName As String, strAddress As String, strCity As String
  Here all the Variables Will Be Declared as String.
Useing Right Kind Of Variables
  As a Programmer You Would know that What Kind Of Calculations or
  What Operations You want to Perform With Different Variables in
  Your Application. If You are Declaring Any Variable Which Would
  Get the Message-Box's Return Value, Then there is No Need to Declare
  It As an Integer or Long. Instead Of that You Can Use Byte(0 to 255) Which
  Falls in the Range of The Message-Box's Returned Value. Using Integer or
  Long over here will Require More Memory than a Byte
  I had also Read Somewhere That If an Integer is Required than Use
  Long Because Long is 32-bit and Integer is 16-bit. Our Operating System
  Is 32-bit So it will be More Convinient for Processor to Work With Long
  Which are 32-bit In place of Integers which are 16-bit.
  One More Thing Use Variable Length Strings Instead of Fixed Length
  Strings. Fixed Length Strings are Slower Compared to the Other.
Another Way Of Declaring Variables
  You can also Declare Variables By their TypeCast
  Eg:- Dim strName As String can Also Be written As
  Dim strName$
  For Integers...... Dim intNumber%
  For Single...... Dim sngNumber!
  For Double...... Dim dblNumber#
  For Long...... Dim lngNumber&
  For Currency...... Dim curNumber@
  You Can Get All the Signs Related To this In Your MSDN Collection
Using Static and Dynamic Variables
  Static variables reside in the memory for the whole Execution time.
  While Dynamic Variables are Procedure Level Variables That are Only
  Created if that Procedure is Entered During Execution. They are Also
  Destroyed When The Execution of that Particular Procedure Ends.
  This Way By Using Dynamic Variables You Create More Robust Applications.
Reclaiming Memory from Variables
  If By any Means You Need to Use Static Variables Then You can
  Reclaim Memory by Destroying if they are not Needed any more.
  Strings can be destroyed using vbNullString keyword
  Eg:- strName = vbNullString Destroys strName Variable
  For Forms That are Loaded In Memory You Can Unload Them
  with Unload (FormName) keyWord
  For Variants You Have To Use Empty KeyWord
  varX = Empty Destroys varX named Variant.
  And Objects Can Be Destroyed By Nothing Keyword
  Set objXYZ = Nothing
