Standard Things In GUI
  There Are also Some Standard Rules If You want to Follow Them.
  First Of all, Users are Familiar With Menus and Tool-Bars So If
  Possible Than Please Include them in you Application.
  There Should always Be a ShortCut Key For Each Menu Item.
  In short Whole Application Should Run With KeyBoard If Mouse
  Is Not Available or it is Broken or Useless etc.
  Now Most Important thing is that Don't Use Controls in
  Such a Way That User Gets Baffled the First Time He Sees the Screen.
  There are many other Things Which I know Nobody Is Going to Follow
  and If I would Write Them Here this Tutorial will Go On and On....

Custom Setup Program
  There are many Installation Creator Programs in Todays Market.
  When We download a New Utility and Start its Installation we get
  The First Impression about that Software. Some Setup Creators are
  are very Good to Look at. They Don't have that Blue Screen which
  Always Comes Up When Installing. You know What I am talking about.
  Okay So Now You Want to also Launch Your Utilities With this Kind Of Setups.
  There is No Need To buy a Setup Creator For this. Just Go to the Directory
  Where You Have Installed Visual Basic. There Will Be a Folder Named 'vb98'
  In there Go to "Wizards\PDWizard\Setup1". Here There Will Be a Project
  Named 'Setup1.vbp' Open It in VB and Modify it. Create a New
  Modified Setup1.exe and Replace the Older-One. Thats it.
  Note:- Make a Back-up of the Original Setup1 Files. Because if the
  New Setup Does Not Works Correctly You can Roll-Back to Original and Try Again.

Other Tips
  Now There Are Few Tips which Should be kept in Mind when Programming
  Never Let your User alone. Means Don't Perform BackGround Operations
  That are too Long. User will Get Bored Just Looking at the Main Screen.
  Put Progress-Bars for showing Progress of Work Done or Display Message Boxes or
  Show Information in Status-Bar and If Nothing than just Beep.
  Don't Use Colors Which can Create Stress to the Users Eyes. Use Light Colors.
  And One Main Thing Don't Over Do all This Or else your Program will Use Much
  More Memory for graphics than its Operations.
  If you want a Plain Background than Use Colors instead of Plain Colored Bitmaps.
  Use Image-Box if the Purpose is only to show the Image.
  Also use Image List Control for Storing the Pictures. It will do Good.
  Don't Use LoadPicture( ) Function For Loading a same Image in more than 1 Picture-Box
  Instead of it Use 'Picture2.Picture = Picture1.Picture' It is More Efficient
  Also Check that AutoRedraw is False Don't Use this Option if Not Necessary
  Reclaim Memory Used By Picture-Boxes by using Cls or Setting its Picture
  Property to Nothing. Thats all For Tips.
